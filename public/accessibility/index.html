accessibility

We want npmx to be usable by as many people as possible.

Our approach

We try to follow the Web Content Accessibility Guidelines (WCAG) 2.2 and use them as a reference when building features. We don't claim full conformance with any level of WCAG — accessibility is a continual process and there is always more work to do.

This site is an open-source, community-driven project. Accessibility improvements are made incrementally as part of our regular development.

What we do

Some of the things we aim to do across the site:

  • —Use semantic HTML and ARIA attributes where appropriate.
  • —Use relative text sizes so you can adjust them in your browser.
  • —Support keyboard navigation throughout the interface.
  • —Respect the prefers-reduced-motion and prefers-color-scheme media queries.
  • —Design with sufficient color contrast in mind.
  • —Ensure essential content is available without JavaScript, though some interactive features require it.

Known limitations

Some parts of the site — particularly third-party content like package READMEs — may not meet accessibility standards. We are working to improve these areas over time.

Feedback

If you encounter an accessibility barrier on npmx, please let us know by opening an issue on our GitHub repository . We take these reports seriously and will do our best to address them.
